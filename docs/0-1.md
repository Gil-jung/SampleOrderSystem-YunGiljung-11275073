# [Phase 0] 사이클 0-1 — pytest 실행 환경 구성

`PLAN.md`의 Phase 0 [0-1] 항목에 대한 tdd 스킬 사이클 기록. RED → GREEN → REVIEW 각 단계를
사람 파트너 승인 후 이 문서에 이어서 기록한다.

## RED

### 목표 (단일 동작)

`src/`에 작성할 프로덕션 코드와 `tests/`에 작성할 테스트 코드를 pytest가 올바르게
인식하도록 스캐폴딩한다.

이 사이클은 "동작(behavior)" 구현이 아니라 설정 작업이므로, tdd 스킬의 예외 규정
("설정 파일은 사람 파트너에게 물어볼 것")에 따라 엄격한 Red→Green이 아니라
**설정 + 최소 확인 테스트 1개**로 진행한다.

### 만들 것

1. 디렉터리: `src/`, `tests/` (빈 디렉터리, 아직 도메인 코드 없음)
2. `pytest.ini` (루트에 신규 생성):
   ```ini
   [pytest]
   testpaths = tests
   pythonpath = src
   ```
   → `tests/`에서 `from model.sample import Sample`처럼 `src/` 하위 모듈을 바로
   import할 수 있게 한다.
3. 확인용 테스트 `tests/test_scaffolding.py`:
   ```python
   def test_pytest_discovers_tests():
       assert True
   ```
   → pytest 실행 시 `tests/` 아래 테스트가 정상적으로 수집·실행됨을 확인하는 용도이며,
   Phase 1부터는 이 파일을 더 이상 확장하지 않는다.

### 검증 방법

```bash
pytest
```
→ `1 passed` 확인.

### 승인 상태

- [x] 사람 파트너 승인 완료 (2026-07-15)

## GREEN

- `pytest.ini` (루트 신규 생성):
  ```ini

  testpaths = tests

  ```
- `tests/test_scaffolding.py`:
