### 목표 (단일 동작)

출고 처리 서브메뉴에서 `2`(출고 실행)를 선택하고 `order_id`를 입력하면 주문이
`RELEASE`로 전환된다.

### 작성할 실패 테스트

`tests/test_main.py`에 아래 테스트를 추가한다.

```python
def test_출고_메뉴에서_출고_실행하면_RELEASE로_전환된다():
    user_input = "\n".join(
        [
            "1",  # 시료 관리
            "1",  # 등록
            "SMP-001",
            "Wafer-A",
            "2.5",
            "0.9",
            "0",
            "2",  # 주문
            "1",  # 예약 (재고 0 -> 승인 시 PRODUCING)
            "SMP-001",
            "홍길동",
            "9",
            "3",  # 승인
            "ORD-0001",
            "0",
            "5",  # 생산 라인
            "3",  # 생산 완료 처리 -> CONFIRMED
            "0",
            "4",  # 출고 처리
            "2",  # 출고 실행
            "ORD-0001",
            "0",
            "3",  # 모니터링
            "1",  # 주문량 조회
            "0",
            "0",
            "",
        ]
    )

    result = subprocess.run(
        [sys.executable, "main.py"],
        input=user_input,
        capture_output=True,
        text=True,
        cwd=SRC_DIR,
    )

    assert result.returncode == 0
    assert result.stderr == ""
    assert "RELEASE: 1" in result.stdout
    assert "CONFIRMED: 0" in result.stdout
```

### 구현 방향 (GREEN 예고 — RED 단계에서는 작성하지 않음)

- `main.py`의 `handle_release_menu`에 `elif choice == "2":` 분기를 추가한다:
  `order_id`를 입력받아 `controller.release_order(order_id)`를 호출하고
  `"출고 완료"`를 출력한다.

### 검증 방법

```bash
pytest tests/test_main.py -v
```

→ `"2"` 분기가 아직 없어 출고가 실행되지 않아, 주문이 여전히 `CONFIRMED`로 남아
`assert "RELEASE: 1" in result.stdout`가 실패하는 것을 확인한다.
