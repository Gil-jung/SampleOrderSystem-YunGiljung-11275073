## RED

### 목표 (단일 동작)

생산 라인 서브메뉴에서 `1`(생산 현황 조회)을 선택하면 현재 생산 중인 주문의 정보
(주문 ID, 실 생산량)가 콘솔에 출력된다.

### 작성할 실패 테스트

`tests/test_main.py`에 아래 테스트를 추가한다.

```python
def test_생산_라인_메뉴에서_생산_현황을_조회하면_실_생산량이_출력된다():
    user_input = "\n".join(
        [
            "1",  # 시료 관리
            "1",  # 등록
            "SMP-001",
            "Wafer-A",
            "2.5",
            "0.9",
            "0",
            "2",  # 주문
            "1",  # 예약 (재고 0 -> 승인 시 PRODUCING)
            "SMP-001",
            "홍길동",
            "9",
            "3",  # 승인
            "ORD-0001",
            "0",
            "5",  # 생산 라인
            "1",  # 생산 현황 조회
            "0",
            "0",
            "",
        ]
    )

    result = subprocess.run(
        [sys.executable, "main.py"],
        input=user_input,
        capture_output=True,
        text=True,
        cwd=SRC_DIR,
    )

    assert result.returncode == 0
    assert result.stderr == ""
    assert "ORD-0001" in result.stdout
    assert "10" in result.stdout  # ceil(9/0.9) = 10 (실 생산량)
```

### 구현 방향 (GREEN 예고 — RED 단계에서는 작성하지 않음)

- `ConsoleView.render_current_production(status)`: `status`가 `None`이면
  `"현재 생산 중인 항목이 없습니다."`, 아니면
  `f"[{status['order_id']}] 실 생산량: {status['actual_production']}, 총 생산 시간: {status['total_production_time']}"`
  형태로 렌더링.
- `main.py`의 `handle_production_menu`에 `elif choice == "1":` 분기를 추가해
  `controller.get_current_production_status()`를 `view.render_current_production(...)`로 출력한다.

### 검증 방법


→ `"1"` 분기가 아직 없어 생산 현황이 출력되지 않아 실패하는 것을 확인한다.
