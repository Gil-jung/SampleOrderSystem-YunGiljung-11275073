# [Phase 10] 사이클 10-5c — 생산 라인 서브메뉴: 생산 완료 처리

`PLAN.md`의 Phase 10 [10-5c] 항목에 대한 tdd 스킬 사이클 기록.

## RED

### 목표 (단일 동작)

메인 메뉴에서 `5`를 선택하면 생산 라인 서브메뉴로 진입하고, `3`(생산 완료 처리)을
선택하면 생산 큐 맨 앞 주문이 `CONFIRMED`로 전환된다.

### 설계 메모 — [5-7]/[9-2] 결정 번복

[5-7]/[9-2]에서는 `complete_production()`을 "시스템 이벤트"로 간주해 콘솔 메뉴에
노출하지 않기로 했었다. 그러나 [10-4a] 진행 중 발견한 대로, 메뉴에 노출하지 않으면
콘솔 UI만으로는 재고를 확보할 방법이 전혀 없어 `CONFIRMED`/`RELEASE` 상태에 도달할
수 없다는 실사용 문제가 있다. 사람 파트너와 논의해 생산 라인 메뉴에 "생산 완료 처리"
명령을 추가하기로 결정했다 (`docs/10-4a.md` 참조).

### 작성할 실패 테스트

`tests/test_main.py`에 아래 테스트를 추가한다.

```python
def test_생산_라인_메뉴에서_생산_완료_처리하면_CONFIRMED로_전환된다():
    user_input = "\n".join(
        [
            "1",  # 시료 관리
            "1",  # 등록
            "SMP-001",
            "Wafer-A",
            "2.5",
            "0.9",
            "0",
            "2",  # 주문
            "1",  # 예약 (재고 0 -> 승인 시 PRODUCING)
            "SMP-001",
            "홍길동",
            "9",
            "3",  # 승인
            "ORD-0001",
            "0",
            "5",  # 생산 라인
            "3",  # 생산 완료 처리
            "0",
            "3",  # 모니터링
            "1",  # 주문량 조회
            "0",
            "0",
            "",
        ]
    )

    result = subprocess.run(
        [sys.executable, "main.py"],
        input=user_input,
        capture_output=True,
        text=True,
        cwd=SRC_DIR,
    )

    assert result.returncode == 0
    assert result.stderr == ""
    assert "CONFIRMED: 1" in result.stdout
    assert "PRODUCING: 0" in result.stdout
```

### 구현 방향 (GREEN 예고 — RED 단계에서는 작성하지 않음)

- `MenuController.complete_current_production()`: `self._order_service.complete_production()`
  그대로 위임 (이미 [5-7]에서 구현된 `OrderService.complete_production()`을 재사용).
- `ConsoleView.render_production_menu()`: "1. 생산 현황 조회 / 2. 대기 큐 조회 /
  3. 생산 완료 처리 / 0. 뒤로가기".
- `main.py`에 `handle_production_menu(controller, view)`를 추가하고 메인 루프
  `choice == "5"`에서 호출한다. 이번 사이클에서는 `3`(생산 완료 처리) 분기만 구현한다
  (`controller.complete_current_production()` 호출 후 `"생산 완료"` 출력).
  `1`/`2`(현황/대기 큐 조회)는 [10-5a]/[10-5b]에서 추가한다.

### 검증 방법

```bash
pytest tests/test_main.py -v
```

→ `main.py`가 아직 `choice == "5"` 분기를 처리하지 않으므로, 생산 완료가 실행되지
않아 주문이 `PRODUCING`으로 남아 `assert "CONFIRMED: 1" in result.stdout`가 실패하는
것을 확인한다.
