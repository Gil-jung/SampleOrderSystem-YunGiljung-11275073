# [Phase 4] 사이클 4-2 — 승인 (재고 충분)

`PLAN.md`의 Phase 4 [4-2] 항목에 대한 tdd 스킬 사이클 기록.

## RED

### 목표 (단일 동작)

재고가 주문 수량 이상인 `RESERVED` 주문을 `OrderService.approve(order_id)`로 승인하면,
주문 상태가 즉시 `CONFIRMED`로 전환되고 시료 재고가 주문 수량만큼 차감된다. (PRD.md 4.3절)

### 설계 메모

- `Sample`은 일반 `dataclass`(불변 아님)이므로 `stock` 필드를 직접 갱신할 수 있다.
  테스트에서 `sample_repository.get(id).stock = 10`처럼 재고를 미리 세팅하는 것은
  [1-5]의 `_make_order`와 동일하게 테스트 셋업(전제 조건 구성)이다.
- 재고 부족 케이스(생산 큐 등록, `PRODUCING` 전환)는 [4-3]에서 다룬다. 이번 사이클은
  "재고 충분" 분기만 구현한다.

### 작성할 실패 테스트

`tests/service/test_order_service.py`에 아래 테스트를 추가한다.

```python
def test_재고가_충분하면_승인_시_즉시_CONFIRMED로_전환되고_재고가_차감된다():
    order_repository = OrderRepository()
    sample_repository = SampleRepository()
    sample_repository.add(
        Sample(sample_id="SMP-001", name="Wafer-A", avg_production_time=2.5, yield_rate=0.9)
    )
    sample_repository.get("SMP-001").stock = 10
    service = OrderService(order_repository, sample_repository)
    order_id = service.reserve(sample_id="SMP-001", customer_name="홍길동", quantity=5)

    service.approve(order_id)

    approved_order = order_repository.get(order_id)
    assert approved_order.status == OrderStatus.CONFIRMED
    assert sample_repository.get("SMP-001").stock == 5
```

### 구현 방향 (GREEN 예고 — RED 단계에서는 작성하지 않음)

- `OrderService.approve(order_id)`:
  1. `order_repository.get(order_id)`로 주문을 조회한다.
  2. `sample_repository.get(order.sample_id)`로 시료를 조회한다.
  3. `sample.stock >= order.quantity`이면:
     - `sample.stock -= order.quantity`
     - `order.transition_to(OrderStatus.CONFIRMED)`
  4. 재고 부족 분기는 이번 사이클에서 구현하지 않는다([4-3]에서 추가).

### 검증 방법


→ `OrderService.approve`가 아직 없으므로 `AttributeError`로 실패하는 것을 확인한다.
