### 목표 (단일 동작)

메인 메뉴에서 `2`를 선택하면 주문 서브메뉴로 진입하고, `1`(예약)을 선택해 시료 ID/
고객명/수량을 입력하면 주문이 예약되고 발급된 `order_id`가 콘솔에 출력된다.

### 작성할 실패 테스트

`tests/test_main.py`에 아래 테스트를 추가한다.

```python
def test_주문_메뉴에서_예약하면_주문ID가_출력된다():
    user_input = "\n".join(
        [
            "1",  # 메인 메뉴 -> 시료 관리
            "1",  # 등록
            "SMP-001",
            "Wafer-A",
            "2.5",
            "0.9",
            "0",  # 시료 관리 메뉴 뒤로가기
            "2",  # 메인 메뉴 -> 주문
            "1",  # 예약
            "SMP-001",
            "홍길동",
            "5",
            "0",  # 주문 메뉴 뒤로가기
            "0",  # 메인 메뉴 종료
            "",
        ]
    )

    result = subprocess.run(
        [sys.executable, "main.py"],
        input=user_input,
        capture_output=True,
        text=True,
        cwd=SRC_DIR,
    )

    assert result.returncode == 0
    assert result.stderr == ""
    assert "ORD-" in result.stdout
```

### 구현 방향 (GREEN 예고 — RED 단계에서는 작성하지 않음)

- `ConsoleView.render_order_menu()`: "1. 예약 / 2. 접수된 목록 조회 / 3. 승인 / 4. 거절 / 0. 뒤로가기".
- `main.py`에 `handle_order_menu(controller, view)` 추가, 메인 루프 `choice == "2"`에서 호출.
  - `1`(예약): `sample_id`/`customer_name`/`quantity`를 입력받아
    `controller.reserve_order(...)`를 호출하고, 반환된 `order_id`를 `f"예약 완료: {order_id}"`로 출력.
  - `0`은 서브메뉴를 빠져나온다.
  - 목록조회/승인/거절 분기는 이번 사이클에서 구현하지 않는다([10-2b]~[10-2d]에서 추가).

### 검증 방법

```bash
pytest tests/test_main.py -v
```

→ `main.py`가 아직 `choice == "2"` 분기를 처리하지 않으므로, 예약이 실행되지 않아
`assert "ORD-" in result.stdout`가 실패하는 것을 확인한다.
