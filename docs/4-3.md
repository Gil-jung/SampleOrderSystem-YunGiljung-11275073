# [Phase 4] 사이클 4-3 — 승인 (재고 부족)

`PLAN.md`의 Phase 4 [4-3] 항목에 대한 tdd 스킬 사이클 기록.

## RED

### 목표 (단일 동작)

재고가 주문 수량보다 부족한 `RESERVED` 주문을 승인하면, 생산 큐에 등록되고 주문 상태가
`PRODUCING`으로 전환된다. (PRD.md 4.3절) 큐 내부 계산(실 생산량, 총 생산 시간 등)은
Phase 5에서 다루고, 이번 사이클은 "큐에 등록되었다"까지만 검증한다.

### 설계 메모 — 최소 `ProductionService` 도입 + 기존 테스트 영향

- 이번 사이클에서 Phase 5의 `ProductionService`를 미리 전부 구현하지 않고, [4-3]이
  필요로 하는 최소 계약만 먼저 만든다: `enqueue(order_id)`(큐에 추가), `list_queue()`
  (현재 큐 내용 조회, FIFO 순서 보장은 리스트 append 순서로 자연히 만족됨). 실 생산량
  계산, 생산 완료 처리 등은 Phase 5 사이클([5-1]~[5-7])에서 별도로 추가한다.
- `OrderService`가 승인 시 생산 큐에 등록하려면 `ProductionService`를 알아야 하므로,
  생성자에 `production_service`를 세 번째 필수 인자로 추가한다:
  `OrderService(order_repository, sample_repository, production_service)`.
  [3-2]에서 `sample_repository`를 추가했을 때와 동일한 이유로, 기존 5개 테스트의
  `OrderService(...)` 생성 호출부도 함께 `ProductionService()`를 전달하도록 조정한다.

### 작성할 실패 테스트

`tests/service/test_order_service.py`

```python
from service.production_service import ProductionService


def test_재고가_부족하면_승인_시_생산_큐에_등록되고_PRODUCING으로_전환된다():
    order_repository = OrderRepository()
    sample_repository = SampleRepository()
    sample_repository.add(
        Sample(sample_id="SMP-001", name="Wafer-A", avg_production_time=2.5, yield_rate=0.9)
    )
    sample_repository.get("SMP-001").stock = 2
    production_service = ProductionService()
    service = OrderService(order_repository, sample_repository, production_service)
    order_id = service.reserve(sample_id="SMP-001", customer_name="홍길동", quantity=5)

    service.approve(order_id)

    approved_order = order_repository.get(order_id)
    assert approved_order.status == OrderStatus.PRODUCING
    assert order_id in production_service.list_queue()
```

기존 5개 테스트도 `OrderService(order_repository, sample_repository, ProductionService())`
형태로 세 번째 인자를 추가한다.

### 구현 방향 (GREEN 예고 — RED 단계에서는 작성하지 않음)

- `src/service/production_service.py` (신규): `ProductionService`
  - `__init__`: 내부 큐 `self._queue = []`
  - `enqueue(order_id)`: 큐에 추가
  - `list_queue()`: 큐 내용을 리스트로 반환 (append 순서 = FIFO 순서)
- `OrderService.__init__`에 `production_service` 인자 추가.
- `OrderService.approve`의 재고 부족(`else`) 분기에서:
  - `order.transition_to(OrderStatus.PRODUCING)`
  - `self._production_service.enqueue(order_id)`

## GREEN

`src/service/production_service.py`를 신규 추가하고, `OrderService`에
`production_service`를 주입해 재고 부족 분기를 구현했다.

```python
# service/production_service.py
class ProductionService:
    def __init__(self):
        self._queue = []

    def enqueue(self, order_id):
        self._queue.append(order_id)

    def list_queue(self):
        return list(self._queue)
```

```python
# service/order_service.py (approve 내부)
if sample.stock >= order.quantity:
    sample.stock -= order.quantity
    order.transition_to(OrderStatus.CONFIRMED)
else:
    order.transition_to(OrderStatus.PRODUCING)
    self._production_service.enqueue(order_id)
```
