# [Phase 8] 사이클 8-5 — 생산 라인 메뉴 흐름

`PLAN.md`의 Phase 8 [8-5] 항목에 대한 tdd 스킬 사이클 기록.

## RED

### 목표 (단일 동작 — 생산 현황/대기 큐 조회 2개 명령의 위임)

`MenuController`의 생산 라인 명령(생산 현황 조회, 대기 큐 조회)이 `ProductionService`로
위임된다. (PRD.md 5절)

### 설계 메모 — 기존 테스트에 영향

`MenuController` 생성자에 `production_service`를 네 번째 필수 인자로 추가한다:
`MenuController(sample_service, order_service, monitoring_service, production_service)`.
기존 9개 테스트의 생성 호출부도 함께 조정한다. `ProductionService.list_queue()`/
`current_status()`는 이미 [5-4]~[5-6]에서 필요한 정보(주문 ID, 실 생산량, 총 생산
시간)를 반환하도록 구현되어 있으므로, Controller는 그대로 위임하기만 하면 된다
(PRD.md "표기할 정보 수준은 자율적으로 결정"에 따라 별도 가공 없이 그대로 노출).

### 작성할 실패 테스트

`tests/controller/test_menu_controller.py`에 아래 테스트를 추가한다.

```python
def test_생산_대기_큐_조회_명령이_ProductionService로_위임된다():
    sample_repository = SampleRepository()
    sample_repository.add(
        Sample(sample_id="SMP-001", name="Wafer-A", avg_production_time=2.5, yield_rate=0.9)
    )
    order_repository = OrderRepository()
    production_service = ProductionService()
    sample_service = SampleService(sample_repository)
    order_service = OrderService(order_repository, sample_repository, production_service)
    monitoring_service = MonitoringService(order_repository, sample_repository)
    controller = MenuController(
        sample_service, order_service, monitoring_service, production_service
    )
    order_id = controller.reserve_order(
        sample_id="SMP-001", customer_name="홍길동", quantity=5
    )
    controller.approve_order(order_id)  # 재고 부족(0) -> PRODUCING, 큐에 등록

    result = controller.get_production_queue()

    assert result == [order_id]


def test_생산_현황_조회_명령이_ProductionService로_위임된다():
    sample_repository = SampleRepository()
    sample_repository.add(
        Sample(sample_id="SMP-001", name="Wafer-A", avg_production_time=2.5, yield_rate=0.9)
    )
    order_repository = OrderRepository()
    production_service = ProductionService()
    sample_service = SampleService(sample_repository)
    order_service = OrderService(order_repository, sample_repository, production_service)
    monitoring_service = MonitoringService(order_repository, sample_repository)
    controller = MenuController(
        sample_service, order_service, monitoring_service, production_service
    )
    order_id = controller.reserve_order(
        sample_id="SMP-001", customer_name="홍길동", quantity=9
    )
    controller.approve_order(order_id)  # 재고 부족(0) -> PRODUCING

    result = controller.get_current_production_status()

    assert result["order_id"] == order_id
    assert result["actual_production"] == 10  # ceil(9/0.9) = 10
```

기존 9개 테스트의 `MenuController(sample_service, order_service, monitoring_service)`도
`MenuController(sample_service, order_service, monitoring_service, production_service)`로
조정한다.

### 구현 방향 (GREEN 예고 — RED 단계에서는 작성하지 않음)

- `MenuController.__init__(self, sample_service, order_service, monitoring_service, production_service)`로 변경.
- `get_production_queue()`: `self._production_service.list_queue()` 그대로 반환.
- `get_current_production_status()`: `self._production_service.current_status()` 그대로 반환.
