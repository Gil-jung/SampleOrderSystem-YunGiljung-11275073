# [Phase 1] 사이클 1-4 — Order 수량 검증

`PLAN.md`의 Phase 1 [1-4] 항목에 대한 tdd 스킬 사이클 기록.

## RED

### 목표 (단일 동작)

`Order` 생성 시 `quantity <= 0`이면 생성이 거부된다(`ValueError`). (PRD.md 3.2절 —
예약 시 입력값인 주문 수량은 자연수여야 한다.)

### 작성할 실패 테스트

`tests/model/test_order.py`에 아래 테스트를 추가한다.

```python
import pytest


@pytest.mark.parametrize("quantity", [0, -1, -10])
def test_수량이_0_이하이면_거부된다(quantity):
    with pytest.raises(ValueError):
        Order(
            sample_id="SMP-001",
            customer_name="홍길동",
            quantity=quantity,
        )
```

### 구현 방향 (GREEN 예고 — RED 단계에서는 작성하지 않음)

- `Order`에 `__post_init__`을 추가해 `quantity <= 0`이면 `ValueError`를 발생시킨다.
- [1-2]에서 `Sample`에 적용한 것과 동일한 패턴(`__post_init__` 검증)을 `Order`에도 적용한다.

### 검증 방법

```bash
pytest tests/model/test_order.py -v
```

→ 현재 `Order`에는 수량 검증이 없으므로 `DID NOT RAISE` 실패가 발생하는 것을 확인한다.
