## RED

### 목표 (단일 동작 — 스모크 테스트)

`src/main.py`가 실행되어 메인 메뉴를 표시하고, 종료 명령("0")을 입력하면 예외 없이
정상 종료(`exit code 0`, `stderr` 빈 문자열)된다. `PoC/DataPersistence`의
`test_main_runs_without_error` 패턴을 따른다.

### 작성할 실패 테스트

`tests/test_main.py` (신규 파일, 루트 레벨 스모크 테스트이므로 `tests/` 바로 아래에 둔다)

```python
import subprocess
import sys
from pathlib import Path

SRC_DIR = Path(__file__).resolve().parent.parent / "src"


def test_main_실행_시_0_입력하면_정상_종료된다():
    result = subprocess.run(
        [sys.executable, "main.py"],
        input="0\n",
        capture_output=True,
        text=True,
        cwd=SRC_DIR,
    )

    assert result.returncode == 0
    assert result.stderr == ""
```

### 구현 방향 (GREEN 예고 — RED 단계에서는 작성하지 않음)

- `src/main.py` (신규): 5개 Repository/Service를 조립해 `MenuController`와
  `ConsoleView`를 생성하고, 메인 메뉴를 표시하는 루프를 돈다.
  - 메뉴 표시: `view.render_main_menu(summary)` (요약 정보는 `controller.list_samples()`
    기반으로 계산).
  - `input()`으로 선택지를 받아 `"0"`이면 종료 메시지를 출력하고 루프를 빠져나온다.
  - 이번 사이클은 "정상 종료"만 검증하므로, 각 메뉴(1~5)의 하위 흐름 구현은 다루지
    않는다 — 잘못된 입력이나 다른 메뉴 번호에 대한 처리는 최소한으로만 둔다.

### 검증 방법

```bash

```

→ `src/main.py`가 아직 없으므로 `subprocess`가 `FileNotFoundError` 또는
0이 아닌 종료 코드로 실패하는 것을 확인한다.
