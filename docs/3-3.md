# [Phase 3] 사이클 3-3 — 0 이하 수량 예약 거부 (통합 확인)

`PLAN.md`의 Phase 3 [3-3] 항목에 대한 tdd 스킬 사이클 기록.

## RED 이전 검토 — 이 사이클도 진짜 RED를 만들 수 없다

목표: `OrderService.reserve`를 통해 예약할 때도 `quantity <= 0`이면 거부된다
(Model의 [1-4] 규칙이 Service를 통해서도 지켜지는지 확인).

`OrderService.reserve`의 현재 구현을 보면:

```python
def reserve(self, sample_id, customer_name, quantity):
    try:
        self._sample_repository.get(sample_id)
    except KeyError:
        raise ValueError(f"unknown sample_id: {sample_id}")

    order = Order(sample_id=sample_id, customer_name=customer_name, quantity=quantity)
    return self._order_repository.add(order)
```

`Order(...)` 생성자는 [1-4]에서 이미 `quantity <= 0`이면 `ValueError`를 발생시키도록
구현되어 있다. `reserve`는 이 생성자를 그대로 호출하므로, 등록된 시료에 대해
0 이하 수량으로 예약을 시도하면 **이미 [1-4] + [3-2]의 조합만으로 거부된다.**

[2-5]와 동일한 이유로, 지금 확인 테스트를 작성하면 처음부터 통과해버려 RED(실패)를
볼 수 없다.

### 제안

- 새로운 프로덕션 코드 변경은 없다.
- 이 통합 지점(Service → Model 경계에서 검증이 실제로 전파되는지)은 회귀 방지 가치가
  있으므로, **RED 없이 확인 테스트 1개만 추가**하고 즉시 통과함을 기록한다.
- `PLAN.md`에는 [3-3]이 [1-4] + [3-2] 구현으로 이미 충족되었다는 메모를 추가한다.

def test_등록된_시료여도_수량이_0_이하이면_예약이_거부된다():
