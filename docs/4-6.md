# [Phase 4] 사이클 4-6 — 주문 거절

`PLAN.md`의 Phase 4 [4-6] 항목에 대한 tdd 스킬 사이클 기록.

## RED

### 목표 (단일 동작)

`RESERVED` 주문을 `OrderService.reject(order_id)`로 거절하면 즉시 `REJECTED`로 전환되고,
재고에는 변화가 없다. (PRD.md 4.3절)

### 작성할 실패 테스트

`tests/service/test_order_service.py`에 아래 테스트를 추가한다.

```python
def test_주문_거절_시_즉시_REJECTED로_전환되고_재고_변화가_없다():
    order_repository = OrderRepository()
    sample_repository = SampleRepository()
    sample_repository.add(
        Sample(sample_id="SMP-001", name="Wafer-A", avg_production_time=2.5, yield_rate=0.9)
    )
    sample_repository.get("SMP-001").stock = 10
    production_service = ProductionService()
    service = OrderService(order_repository, sample_repository, production_service)
    order_id = service.reserve(sample_id="SMP-001", customer_name="홍길동", quantity=5)

    service.reject(order_id)

    rejected_order = order_repository.get(order_id)
    assert rejected_order.status == OrderStatus.REJECTED
    assert sample_repository.get("SMP-001").stock == 10
```

### 구현 방향 (GREEN 예고 — RED 단계에서는 작성하지 않음)

- `OrderService.reject(order_id)`:
  1. `order_repository.get(order_id)`로 주문을 조회한다.
  2. `order.transition_to(OrderStatus.REJECTED)`를 호출한다.
- 재고는 건드리지 않는다 — `RESERVED` 상태에서는 아직 재고가 차감된 적이 없으므로
  (재고 차감은 `approve`의 "재고 충분" 분기에서만 일어남) 별도 처리가 필요 없다.
- 존재하지 않는 주문 거절, 이미 처리된 주문 거절 등 예외 케이스는 이 사이클 범위 밖이다
  ([4-7]에서 "이미 처리된 주문 거절"을 다룬다).
