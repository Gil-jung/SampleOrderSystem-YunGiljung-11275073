# [Phase 10] 사이클 10-3a — 모니터링 서브메뉴: 주문량 조회

`PLAN.md`의 Phase 10 [10-3a] 항목에 대한 tdd 스킬 사이클 기록.

## RED

### 목표 (단일 동작)

메인 메뉴에서 `3`을 선택하면 모니터링 서브메뉴로 진입하고, `1`(주문량 조회)을
선택하면 상태별 주문 수가 콘솔에 출력된다.

### 작성할 실패 테스트

`tests/test_main.py`에 아래 테스트를 추가한다.

```python
def test_모니터링_메뉴에서_주문량_조회_시_상태별_건수가_출력된다():
    user_input = "\n".join(
        [
            "1",  # 시료 관리
            "1",  # 등록
            "SMP-001",
            "Wafer-A",
            "2.5",
            "0.9",
            "0",
            "2",  # 주문
            "1",  # 예약
            "SMP-001",
            "홍길동",
            "5",
            "0",
            "3",  # 모니터링
            "1",  # 주문량 조회
            "0",
            "0",
            "",
        ]
    )

    result = subprocess.run(
        [sys.executable, "main.py"],
        input=user_input,
        capture_output=True,
        text=True,
        cwd=SRC_DIR,
    )

    assert result.returncode == 0
    assert result.stderr == ""
    assert "RESERVED: 1" in result.stdout
```

### 구현 방향 (GREEN 예고 — RED 단계에서는 작성하지 않음)

- `ConsoleView`에 추가:
  - `render_monitoring_menu()`: "1. 주문량 조회 / 2. 재고량 조회 / 0. 뒤로가기".
  - `render_order_counts(counts)`: `{"RESERVED": 1, "CONFIRMED": 0, ...}` 형태의 `dict`를
    `"RESERVED: 1"`처럼 한 줄씩 렌더링.
- `main.py`에 `handle_monitoring_menu(controller, view)`를 추가하고, 메인 루프
  `choice == "3"`에서 호출한다.
  - `1`(주문량 조회): `controller.get_order_counts()`를 `view.render_order_counts(...)`로 출력.
  - `2`(재고량 조회)는 이번 사이클에서 다루지 않는다 ([10-3b]에서 추가).
  - `0`은 서브메뉴를 빠져나온다.

### 검증 방법

```bash
pytest tests/test_main.py -v
```

→ `main.py`가 아직 `choice == "3"` 분기를 처리하지 않으므로, 주문량이 출력되지 않아
`assert "RESERVED: 1" in result.stdout`가 실패하는 것을 확인한다.
