# [Phase 1] 사이클 1-2 — Sample 수율 범위 검증

`PLAN.md`의 Phase 1 [1-2] 항목에 대한 tdd 스킬 사이클 기록.

## RED

### 목표 (단일 동작)

`Sample` 생성 시 `yield_rate`가 0~1 범위를 벗어나면 생성이 거부된다(`ValueError`).
(PRD.md 2.1절 — 수율은 정상 생산 시료 수 / 총 생산 시료 수이므로 0~1 사이 값이어야 한다.)

### 작성할 실패 테스트

`tests/model/test_sample.py`에 아래 테스트를 추가한다.

```python
import pytest


@pytest.mark.parametrize("yield_rate", [-0.1, 1.1, -1, 2])
def test_수율이_0에서_1_범위를_벗어나면_거부된다(yield_rate):
    with pytest.raises(ValueError):
        Sample(
            sample_id="SMP-001",
            name="Wafer-A",
            avg_production_time=2.5,
            yield_rate=yield_rate,
        )
```

경계값 0과 1은 정상 허용되어야 하며, 이는 [1-1]에서 이미 `yield_rate=0.9`로 정상 생성이
검증되어 있으므로 이번 사이클에서 별도 테스트를 추가하지 않는다. (동일 동작의 반례이므로
parametrize 하나로 묶는다 — "같은 동작 규칙, 여러 입력"에 해당.)

### 구현 방향 (GREEN 예고 — RED 단계에서는 작성하지 않음)

- `Sample`은 `dataclass`를 유지하되 `__post_init__`에서 `0 <= yield_rate <= 1`을 검증한다.
- 범위를 벗어나면 `ValueError`를 발생시킨다. (전용 예외 클래스는 아직 두지 않는다 — YAGNI.
  추후 Order 쪽에서도 전용 예외(`OrderError`)가 필요해지는 시점에 Sample 쪽도 재검토한다.)

### 검증 방법

```bash
pytest tests/model/test_sample.py -v
```

→ 현재 `Sample`에는 범위 검증이 없으므로, `pytest.raises(ValueError)`가 예외를 잡지 못해
`DID NOT RAISE` 실패가 발생하는 것을 확인한다.
