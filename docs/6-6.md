## RED 이전 검토 — 이 사이클도 진짜 RED를 만들 수 없다

목표: 미출고(`CONFIRMED`) 수요가 없는 시료는 재고 수량과 무관하게 `"여유"`로 판정된다
(`0 >= 0` 경계 케이스 확인).

`stock_status`의 현재 구현을 보면:

```python
if stock >= demand:
    return "여유"
elif stock > 0:
    return "부족"
else:
    return "고갈"
```

`demand`가 0이면(해당 시료에 `CONFIRMED` 주문이 없으면), `stock`이 0이든 양수든
`stock >= 0`은 항상 참이므로 **첫 번째 분기(`"여유"`)에서 이미 걸러진다.** 재고가
0인 경우조차 "고갈"이 아니라 "여유"로 판정되는 것이 바로 이 경계 케이스가 의도한
동작이다. [2-5]/[3-3]/[4-5]/[4-7]/[5-4]/[6-2]와 동일한 이유로, 지금 확인 테스트를
작성하면 처음부터 통과해버려 RED(실패)를 볼 수 없다.

### 제안

- 새로운 프로덕션 코드 변경은 없다.
- 이 경계 케이스는 요구사항으로 명시된 만큼 회귀 방지 가치가 있으므로, **RED 없이
  확인 테스트 1개만 추가**하고 즉시 통과함을 기록한다.
- `PLAN.md`에는 [6-6]이 [6-3]~[6-5]의 `if/elif/else` 구조만으로 이미 충족되었다는
  메모를 추가한다.

```python
def test_미출고_수요가_없으면_재고가_0이어도_여유로_판정한다():
    order_repository = OrderRepository()
    sample_repository = SampleRepository()
    sample_repository.add(
        Sample(sample_id="SMP-001", name="Wafer-A", avg_production_time=2.5, yield_rate=0.9)
    )
    # 등록 직후 재고 0, CONFIRMED 주문 없음(수요 0)
    monitoring_service = MonitoringService(order_repository, sample_repository)

    assert monitoring_service.stock_status("SMP-001") == "여유"
```
