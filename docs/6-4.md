# [Phase 6] 사이클 6-4 — 시료별 재고 상태 판정 (부족)

`PLAN.md`의 Phase 6 [6-4] 항목에 대한 tdd 스킬 사이클 기록.

## RED

### 목표 (단일 동작)

재고가 0보다 크지만 미출고(`CONFIRMED`) 수요 합보다 적으면
`MonitoringService.stock_status(sample_id)`가 `"부족"`을 반환한다. (PRD.md 4.4절)

### 작성할 실패 테스트

`tests/service/test_monitoring_service.py`에 아래 테스트를 추가한다.

```python
def test_재고가_0보다_크고_미출고_수요보다_적으면_부족으로_판정한다():
    order_repository = OrderRepository()
    sample_repository = SampleRepository()
    sample_repository.add(
        Sample(sample_id="SMP-001", name="Wafer-A", avg_production_time=2.5, yield_rate=0.9)
    )
    sample_repository.get("SMP-001").stock = 10
    order_service = OrderService(order_repository, sample_repository, ProductionService())
    monitoring_service = MonitoringService(order_repository, sample_repository)

    order_id = order_service.reserve(sample_id="SMP-001", customer_name="홍길동", quantity=8)
    order_service.approve(order_id)  # 재고 충분 -> CONFIRMED, stock 10 -> 2, 수요 8

    assert monitoring_service.stock_status("SMP-001") == "부족"
```

### 구현 방향 (GREEN 예고 — RED 단계에서는 작성하지 않음)

- `stock_status`에 `elif stock > 0: return "부족"` 분기를 추가한다
  (앞선 `if stock >= demand` 분기에서 이미 "여유" 케이스는 걸러졌으므로, 이 시점에는
  `stock < demand`가 보장된다).

### 검증 방법

```bash
pytest tests/service/test_monitoring_service.py -v
```

→ 현재 `stock_status`는 "여유" 분기 외에는 아무것도 반환하지 않으므로(`None`),
`assert None == "부족"`으로 실패하는 것을 확인한다.
