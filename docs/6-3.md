# [Phase 6] 사이클 6-3 — 시료별 재고 상태 판정 (여유)

`PLAN.md`의 Phase 6 [6-3] 항목에 대한 tdd 스킬 사이클 기록.

## RED

### 목표 (단일 동작)

재고가 해당 시료의 미출고(`CONFIRMED`) 수요 합 이상이면 `MonitoringService.stock_status(sample_id)`가
`"여유"`를 반환한다. (PRD.md 4.4절)

### 설계 메모 — 기존 테스트에 영향

재고 상태 판정에는 시료 재고와 해당 시료의 `CONFIRMED` 주문 수량 합이 모두 필요하다.
`MonitoringService`가 `sample_repository`도 알아야 하므로, 생성자에 두 번째 필수 인자로
추가한다: `MonitoringService(order_repository, sample_repository)`. [3-2]/[4-3]에서
생성자에 새 의존성을 추가했을 때와 동일한 이유로, [6-1]/[6-2]에서 작성한 기존 2개
테스트의 `MonitoringService(...)` 생성 호출부도 `sample_repository`를 함께 전달하도록
조정한다.

### 작성할 실패 테스트

`tests/service/test_monitoring_service.py`

```python
def test_재고가_미출고_수요_이상이면_여유로_판정한다():
    order_repository = OrderRepository()
    sample_repository = SampleRepository()
    sample_repository.add(
        Sample(sample_id="SMP-001", name="Wafer-A", avg_production_time=2.5, yield_rate=0.9)
    )
    sample_repository.get("SMP-001").stock = 10
    order_service = OrderService(order_repository, sample_repository, ProductionService())
    monitoring_service = MonitoringService(order_repository, sample_repository)

    order_id = order_service.reserve(sample_id="SMP-001", customer_name="홍길동", quantity=5)
    order_service.approve(order_id)  # 재고 충분 -> CONFIRMED, stock 10 -> 5

    assert monitoring_service.stock_status("SMP-001") == "여유"
```

승인 후 재고는 5(10-5)이고, 미출고 수요(해당 시료의 `CONFIRMED` 주문 수량 합)도 5이므로
`재고 >= 수요` 경계값(같음)이 "여유"로 판정되는지도 함께 확인한다.

기존 [6-1]/[6-2] 테스트의 `MonitoringService(order_repository)` 호출도
`MonitoringService(order_repository, sample_repository)`로 조정한다.

### 구현 방향 (GREEN 예고 — RED 단계에서는 작성하지 않음)

- `MonitoringService.__init__(self, order_repository, sample_repository)`로 변경.
- `stock_status(sample_id)`:
  1. `demand = sum(order.quantity for order in self._order_repository.list() if order.sample_id == sample_id and order.status == OrderStatus.CONFIRMED)`
  2. `stock = self._sample_repository.get(sample_id).stock`
  3. 이번 사이클에서는 "여유"(`stock >= demand`) 분기만 구현한다. "부족"/"고갈" 분기는
     [6-4]/[6-5]에서 추가한다.
