# [Phase 2] 사이클 2-4 — 이름으로 시료 검색

`PLAN.md`의 Phase 2 [2-4] 항목에 대한 tdd 스킬 사이클 기록.

## RED

### 목표 (단일 동작)

`SampleService.search(name)`이 이름에 `name`이 부분 일치하는 시료만 반환한다. (PRD.md 4.1절)

### 작성할 실패 테스트

`tests/service/test_sample_service.py`에 아래 테스트를 추가한다.

```python
def test_이름으로_시료를_검색하면_부분_일치하는_시료만_반환된다():
    repository = SampleRepository()
    service = SampleService(repository)
    service.register(
        sample_id="SMP-001", name="Wafer-A", avg_production_time=2.5, yield_rate=0.9
    )
    service.register(
        sample_id="SMP-002", name="Wafer-B", avg_production_time=1.0, yield_rate=0.8
    )
    service.register(
        sample_id="SMP-003", name="Other-C", avg_production_time=1.0, yield_rate=0.8
    )

    results = service.search("Wafer")

    ids = {sample.sample_id for sample in results}
    assert ids == {"SMP-001", "SMP-002"}
```

### 구현 방향 (GREEN 예고 — RED 단계에서는 작성하지 않음)

- `SampleService.search(name)`: `list_all()` 결과에서 `name in sample.name`인 항목만 필터링해
  반환한다. Repository를 변경하지 않고 이미 있는 `list_all()`을 재사용한다.
- 빈 검색 결과(일치 없음) 케이스는 [2-5]에서 별도로 다룬다.

### 검증 방법

```bash
pytest tests/service/test_sample_service.py -v
```

→ `SampleService.search`가 아직 없으므로 `AttributeError`로 실패하는 것을 확인한다.

def search(self, name):
    return [sample for sample in self.list_all() if name in sample.name]
