## RED

### 목표 (단일 동작)

`Order(sample_id, customer_name, quantity)`로 생성하면 상태가 `RESERVED`이다.
(PRD.md 3절 — 예약 시점에는 상태가 `RESERVED`.)

`order_id`, `created_at`, `updated_at`은 PRD.md 2.2절 데이터 모델에 포함되어 있으나,
이번 사이클의 테스트가 요구하지 않으므로 지금 추가하지 않는다(YAGNI). 이후 사이클
(Repository/Service 도입 시점)에서 실제로 필요해지면 그때 실패하는 테스트로 추가한다.

### 작성할 실패 테스트

`tests/model/test_order.py` (신규 파일)

```python
from model.order import Order, OrderStatus


def test_order_생성_시_초기_상태는_RESERVED이다():
    order = Order(
        sample_id="SMP-001",
        customer_name="홍길동",
        quantity=10,
    )

    assert order.status == OrderStatus.RESERVED
    assert order.sample_id == "SMP-001"
    assert order.customer_name == "홍길동"
    assert order.quantity == 10
```

### 구현 방향 (GREEN 예고 — RED 단계에서는 작성하지 않음)

- `src/model/order.py`에 `OrderStatus` `Enum`을 정의한다. 이번 사이클에서는 `RESERVED` 멤버만
  추가한다. 나머지 상태(`REJECTED`, `PRODUCING`, `CONFIRMED`, `RELEASE`)는 [1-5] 상태 전이
  규칙 사이클에서 실패하는 테스트와 함께 추가한다.
- `Order`를 `dataclass`로 정의: `sample_id: str`, `customer_name: str`, `quantity: int`,
  `status: OrderStatus = OrderStatus.RESERVED` (기본값으로 초기 상태 보장).
