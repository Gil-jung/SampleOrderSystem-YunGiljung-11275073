# [Phase 4] 사이클 4-7 — 이미 처리된 주문 거절 시도 거부 (통합 확인)

`PLAN.md`의 Phase 4 [4-7] 항목에 대한 tdd 스킬 사이클 기록.

## RED 이전 검토 — 이 사이클도 진짜 RED를 만들 수 없다

목표: `RESERVED`가 아닌 주문(이미 승인/거절된 주문)에 `OrderService.reject`를 호출하면
`OrderError`로 거부된다.

`reject`의 현재 구현은 `order.transition_to(OrderStatus.REJECTED)`만 호출한다.
[1-5]의 전이 테이블에서 `REJECTED`로 갈 수 있는 출발 상태는 `RESERVED` 하나뿐이므로,
이미 `CONFIRMED`/`PRODUCING`/`REJECTED`/`RELEASE`인 주문에 `reject`를 호출하면
**이미 `OrderError`로 거부되고 있다.** [4-5]와 완전히 동일한 이유로, 지금 확인
테스트를 작성하면 처음부터 통과해버려 RED(실패)를 볼 수 없다.

### 제안

- 새로운 프로덕션 코드 변경은 없다.
- Service → Model 경계에서 이 방어가 실제로 전파되는지는 회귀 방지 가치가 있으므로,
  **RED 없이 확인 테스트 1개만 추가**하고 즉시 통과함을 기록한다.
- `PLAN.md`에는 [4-7]이 [1-5]의 전이 테이블만으로 이미 충족되었다는 메모를 추가한다.

### 확인 테스트 (참고용 — RED 없이 바로 통과 예정)

```python
def test_이미_승인된_주문을_거절하면_OrderError로_거부된다():
    order_repository = OrderRepository()
    sample_repository = SampleRepository()
    sample_repository.add(
        Sample(sample_id="SMP-001", name="Wafer-A", avg_production_time=2.5, yield_rate=0.9)
    )
    sample_repository.get("SMP-001").stock = 10
    production_service = ProductionService()
    service = OrderService(order_repository, sample_repository, production_service)
    order_id = service.reserve(sample_id="SMP-001", customer_name="홍길동", quantity=5)
    service.approve(order_id)  # CONFIRMED로 전환됨

    with pytest.raises(OrderError):
        service.reject(order_id)
```
