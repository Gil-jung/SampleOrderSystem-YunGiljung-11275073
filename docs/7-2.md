# [Phase 7] 사이클 7-2 — CONFIRMED가 아닌 주문 출고 시도 거부 (통합 확인)

`PLAN.md`의 Phase 7 [7-2] 항목에 대한 tdd 스킬 사이클 기록.

## RED 이전 검토 — 이 사이클도 진짜 RED를 만들 수 없다

목표: `CONFIRMED`가 아닌 주문에 `ReleaseService.release`를 호출하면 `OrderError`로 거부된다.

`release`의 현재 구현은 `order.transition_to(OrderStatus.RELEASE)`만 호출한다.
[1-5]의 전이 테이블에서 `RELEASE`로 갈 수 있는 출발 상태는 `CONFIRMED` 하나뿐이므로,
`RESERVED`/`PRODUCING`/`REJECTED`/`RELEASE`(이미 출고됨) 상태의 주문에 `release`를
호출하면 **이미 `OrderError`로 거부되고 있다.** [4-5]/[4-7]와 완전히 동일한 이유로,
지금 확인 테스트를 작성하면 처음부터 통과해버려 RED(실패)를 볼 수 없다.

### 제안

- 새로운 프로덕션 코드 변경은 없다.
- Service → Model 경계에서 이 방어가 실제로 전파되는지는 회귀 방지 가치가 있으므로,
  **RED 없이 확인 테스트 1개만 추가**하고 즉시 통과함을 기록한다.
- `PLAN.md`에는 [7-2]가 [1-5]의 전이 테이블만으로 이미 충족되었다는 메모를 추가한다.

### 확인 테스트 (참고용 — RED 없이 바로 통과 예정)

```python
def test_CONFIRMED가_아닌_주문을_출고하면_OrderError로_거부된다():
    order_repository = OrderRepository()
    sample_repository = SampleRepository()
    sample_repository.add(
        Sample(sample_id="SMP-001", name="Wafer-A", avg_production_time=2.5, yield_rate=0.9)
    )
    order_service = OrderService(order_repository, sample_repository, ProductionService())
    release_service = ReleaseService(order_repository)

    order_id = order_service.reserve(sample_id="SMP-001", customer_name="홍길동", quantity=1)
    # 아직 RESERVED 상태 (승인 전)

    with pytest.raises(OrderError):
        release_service.release(order_id)
```
