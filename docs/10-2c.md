# [Phase 10] 사이클 10-2c — 주문 서브메뉴: 승인

`PLAN.md`의 Phase 10 [10-2c] 항목에 대한 tdd 스킬 사이클 기록.

## RED

### 목표 (단일 동작)

주문 서브메뉴에서 `3`(승인)을 선택하고 `order_id`를 입력하면 주문이 승인되고,
승인 완료 후 접수된 목록 조회에서 더 이상 표시되지 않는다.

### 작성할 실패 테스트

`tests/test_main.py`에 아래 테스트를 추가한다.

```python
def test_주문_메뉴에서_승인하면_접수된_목록에서_사라진다():
    user_input = "\n".join(
        [
            "1",  # 시료 관리
            "1",  # 등록
            "SMP-001",
            "Wafer-A",
            "2.5",
            "0.9",
            "0",
            "2",  # 주문
            "1",  # 예약
            "SMP-001",
            "홍길동",
            "5",
            "3",  # 승인
            "ORD-0001",
            "2",  # 접수된 목록 조회
            "0",
            "0",
            "",
        ]
    )

    result = subprocess.run(
        [sys.executable, "main.py"],
        input=user_input,
        capture_output=True,
        text=True,
        cwd=SRC_DIR,
    )

    assert result.returncode == 0
    assert result.stderr == ""
    assert "주문이 없습니다." in result.stdout
```

주문 ID는 [3-1]에서 정한 순번 발급 규칙(`OrderRepository`가 `"ORD-0001"`부터 발급)에
따라 이 시나리오에서는 항상 `"ORD-0001"`이므로 테스트에서 하드코딩해도 안전하다.

### 구현 방향 (GREEN 예고 — RED 단계에서는 작성하지 않음)

- `main.py`의 `handle_order_menu`에 `elif choice == "3":` 분기를 추가한다:
  `order_id`를 입력받아 `controller.approve_order(order_id)`를 호출하고
  `"승인 완료"`를 출력한다.

### 검증 방법

```bash
pytest tests/test_main.py -v
```

→ `"3"` 분기가 아직 없어 주문이 여전히 `RESERVED`로 남아, 접수된 목록 조회에
"주문이 없습니다."가 아니라 주문 내역이 출력되어 실패하는 것을 확인한다.
