# [Phase 1] 사이클 1-5 — OrderStatus 전이 규칙

`PLAN.md`의 Phase 1 [1-5] 항목에 대한 tdd 스킬 사이클 기록.

## 참고: [1-6] "종결 상태 보호"를 이 사이클에 통합

`PLAN.md`는 [1-5](허용된 전이만 성공)와 [1-6](`REJECTED`/`RELEASE`는 전이 불가)을 별도
사이클로 나눠뒀다. 그러나 상태 전이는 "허용된 전이 목록에 있는지"를 확인하는 **하나의
전이 테이블**로 구현되며, 종결 상태 보호는 이 테이블에서 `REJECTED`/`RELEASE`의
허용 전이 목록이 비어있다는 사실의 자연스러운 결과다. 즉 [1-5]의 구현을 마치고 나면
[1-6]의 테스트는 이미 통과하는 상태가 되어 "실패하는 테스트를 먼저 본다"는 RED 원칙을
지킬 수 없다.

따라서 두 사이클을 하나로 합쳐, 유효한 전이 5종 + 무효한 전이(종결 상태 포함) 다수를
한 사이클에서 함께 검증한다. `PLAN.md` 5절의 [1-6] 항목은 이 사이클로 흡수되었다는
메모를 추가할 예정이다 (사람 파트너 승인 시).

## RED

### 목표 (두 가지 동작 — 같은 전이 테이블의 성공/실패 경로)

`Order.transition_to(new_status)`:
1. 허용된 전이(`RESERVED→CONFIRMED`, `RESERVED→PRODUCING`, `RESERVED→REJECTED`,
   `PRODUCING→CONFIRMED`, `CONFIRMED→RELEASE`)는 상태를 성공적으로 변경한다.
2. 그 외 모든 전이(종결 상태 `REJECTED`/`RELEASE`에서의 전이 포함)는 `OrderError`로 거부되고
   상태는 변경되지 않는다.

### 작성할 실패 테스트

`tests/model/test_order.py`에 아래 테스트를 추가한다.

```python
from model.order import Order, OrderError, OrderStatus


def _make_order(status):
    order = Order(sample_id="SMP-001", customer_name="홍길동", quantity=1)
    order.status = status
    return order


@pytest.mark.parametrize(
    "from_status, to_status",
    [
        (OrderStatus.RESERVED, OrderStatus.CONFIRMED),
        (OrderStatus.RESERVED, OrderStatus.PRODUCING),
        (OrderStatus.RESERVED, OrderStatus.REJECTED),
        (OrderStatus.PRODUCING, OrderStatus.CONFIRMED),
        (OrderStatus.CONFIRMED, OrderStatus.RELEASE),
    ],
)
def test_허용된_상태_전이는_성공한다(from_status, to_status):
    order = _make_order(from_status)

    order.transition_to(to_status)

    assert order.status == to_status


@pytest.mark.parametrize(
    "from_status, to_status",
    [
        (OrderStatus.RESERVED, OrderStatus.RELEASE),
        (OrderStatus.CONFIRMED, OrderStatus.PRODUCING),
        (OrderStatus.CONFIRMED, OrderStatus.REJECTED),
        (OrderStatus.PRODUCING, OrderStatus.RESERVED),
        (OrderStatus.PRODUCING, OrderStatus.REJECTED),
        (OrderStatus.REJECTED, OrderStatus.CONFIRMED),
        (OrderStatus.REJECTED, OrderStatus.RELEASE),
        (OrderStatus.RELEASE, OrderStatus.CONFIRMED),
        (OrderStatus.RELEASE, OrderStatus.REJECTED),
    ],
)
def test_허용되지_않은_상태_전이는_OrderError로_거부된다(from_status, to_status):
    order = _make_order(from_status)

    with pytest.raises(OrderError):
        order.transition_to(to_status)

    assert order.status == from_status
```

`_make_order`에서 `order.status = status`로 시작 상태를 강제 설정하는 것은 테스트
셋업(전제 조건 구성)이며, `transition_to`의 동작 자체를 우회하는 것이 아니므로
안티패턴이 아니다.

### 구현 방향 (GREEN 예고 — RED 단계에서는 작성하지 않음)

- `src/model/order.py`에 `OrderStatus`의 나머지 멤버(`REJECTED`, `PRODUCING`, `CONFIRMED`,
  `RELEASE`)를 추가한다.
- `OrderError(Exception)`을 정의한다.
- 모듈 수준 상수 `_ALLOWED_TRANSITIONS: dict[OrderStatus, set[OrderStatus]]`로 허용 전이
  테이블을 정의하고, `Order.transition_to(new_status)`가 이 테이블을 조회해 허용되지
  않으면 `OrderError`를 발생시키고, 허용되면 `self.status = new_status`로 갱신한다.

### 검증 방법

```bash
pytest tests/model/test_order.py -v
```

→ `transition_to` 메서드와 `OrderError`, 나머지 `OrderStatus` 멤버가 아직 없으므로
`AttributeError` 또는 `ImportError`로 실패하는 것을 확인한다.
