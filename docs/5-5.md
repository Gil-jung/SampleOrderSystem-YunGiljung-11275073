# [Phase 5] 사이클 5-5 — 대기 주문 목록 조회

`PLAN.md`의 Phase 5 [5-5] 항목에 대한 tdd 스킬 사이클 기록.

## 검토 — 이 사이클은 [5-4]로 이미 완전히 검증되어 있다

목표: `ProductionService.list_queue()`가 큐에 남은 주문을 FIFO 순서 그대로 반환한다.

[5-4]에서 작성한 확인 테스트(`test_여러_주문이_승인되면_등록_순서대로_큐에_쌓인다`,
`tests/service/test_order_service.py`)가 이미 정확히 이 동작을 검증하고 있다:

```python
assert production_service.list_queue() == [first_id, second_id]
```

이는 "여러 주문이 큐에 등록되는 순서"([5-4])와 "`list_queue()`가 그 순서를 그대로
반환하는 것"([5-5])이 사실상 하나의 관찰 지점이기 때문이다 — 큐에 무엇이 어떤
순서로 쌓였는지는 `list_queue()`를 통해서만 확인할 수 있으므로, 두 요구사항을
분리해서 각각 테스트할 수 없다.

### 결론

- 새로운 프로덕션 코드 변경도, 새로운 확인 테스트도 필요하지 않다.
- `PLAN.md`에 [5-5]가 [5-4]의 확인 테스트로 이미 완전히 커버되었다는 메모만 추가한다.
