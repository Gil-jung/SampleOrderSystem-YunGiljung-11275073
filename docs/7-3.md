# [Phase 7] 사이클 7-3 — 존재하지 않는 주문 출고 시도

`PLAN.md`의 Phase 7 [7-3] 항목에 대한 tdd 스킬 사이클 기록.

## RED

### 목표 (단일 동작)

존재하지 않는 `order_id`로 `ReleaseService.release`를 호출하면 `ValueError`가 발생한다.
(PRD.md 4.6절)

### 설계 메모

[4-4]에서 `OrderService.approve`가 `order_repository`의 `KeyError`를 `ValueError`로
변환했던 것과 동일하게, `ReleaseService.release`도 Repository의 저수준 예외를
노출하지 않고 `ValueError`로 통일한다. 현재 코드는 `order_repository.get(order_id)`를
그대로 호출해 `KeyError`가 새어나오므로, `ValueError`를 기대하는 테스트는 진짜로
실패한다 (예외 타입이 다르기 때문 — [4-4]와 동일한 패턴).

### 작성할 실패 테스트

`tests/service/test_release_service.py`에 아래 테스트를 추가한다.

```python
def test_존재하지_않는_주문을_출고하면_예외가_발생한다():
    order_repository = OrderRepository()
    release_service = ReleaseService(order_repository)

    with pytest.raises(ValueError):
        release_service.release("UNKNOWN-ORDER-ID")
```

### 구현 방향 (GREEN 예고 — RED 단계에서는 작성하지 않음)

- `ReleaseService.release`에서 `order_repository.get(order_id)` 호출을
  `try/except KeyError`로 감싸 `ValueError`로 변환한다.

### 검증 방법

```bash
pytest tests/service/test_release_service.py -v
```

→ `release`가 `KeyError`를 그대로 던지므로, `pytest.raises(ValueError)` 블록 안에서
처리되지 않은 `KeyError`가 발생해 실패하는 것을 확인한다 (오타가 아니라 예외 타입
불일치로 인한 실패).
