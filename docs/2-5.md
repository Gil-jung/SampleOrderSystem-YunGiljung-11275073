# [Phase 2] 사이클 2-5 — 빈 검색 결과

`PLAN.md`의 Phase 2 [2-5] 항목에 대한 tdd 스킬 사이클 기록.

## RED 이전 검토 — 이 사이클은 진짜 RED를 만들 수 없다

목표: 일치하는 시료가 없으면 `SampleService.search(name)`이 빈 목록을 반환한다(예외 아님).

그런데 [2-4]에서 이미 구현한 코드를 보면:

```python
def search(self, name):
    return [sample for sample in self.list_all() if name in sample.name]
```

리스트 컴프리헨션은 일치하는 항목이 하나도 없으면 이미 자연스럽게 빈 리스트 `[]`를
반환하고 예외를 던지지 않는다. 즉 "빈 검색 결과 → 빈 목록, 예외 아님"이라는 동작은
[2-4]의 구현만으로 **이미 충족되어 있다.**

이 상태에서 확인 테스트를 작성하면 [1-5]/[1-6] 때와 동일한 문제가 생긴다 — 테스트가
처음부터 통과해버려 "실패하는 것을 먼저 본다"는 철의 법칙을 지킬 수 없다
(`tdd` 스킬: "테스트가 통과한다고? 이미 존재하는 동작을 검증하고 있는 것이다").

### 제안

- 새로운 프로덕션 코드 변경은 없다 ([2-4]의 구현이 이미 요구사항을 충족).
- 다만 이 동작은 회귀 방지 차원에서 문서화해둘 가치가 있으므로, **RED 없이 확인
  테스트 1개만 추가**하고 즉시 통과함을 기록한다. (Phase 0의 스캐폴딩 사이클과
  유사하게, "새 동작 구현"이 아니라 "이미 있는 동작의 확인"이므로 Red→Green 사이클을
  강제하지 않는다.)
- `PLAN.md`에는 [2-5]가 [2-4] 구현으로 이미 충족되었다는 메모를 추가한다.
