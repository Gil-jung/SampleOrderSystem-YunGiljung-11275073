# [Phase 2] 사이클 2-2 — 중복 ID 등록 거부

`PLAN.md`의 Phase 2 [2-2] 항목에 대한 tdd 스킬 사이클 기록.

## RED

### 목표 (단일 동작)

이미 존재하는 `sample_id`로 `SampleService.register`를 다시 호출하면 거부된다(`ValueError`).
(PRD.md 4.1절 — "시스템에 등록된 시료만 주문 가능" 하려면 시료 ID가 고유해야 한다.)

### 작성할 실패 테스트

`tests/service/test_sample_service.py`에 아래 테스트를 추가한다.

```python
import pytest


def test_이미_존재하는_시료_ID로_등록하면_거부된다():
    repository = SampleRepository()
    service = SampleService(repository)
    service.register(
        sample_id="SMP-001",
        name="Wafer-A",
        avg_production_time=2.5,
        yield_rate=0.9,
    )

    with pytest.raises(ValueError):
        service.register(
            sample_id="SMP-001",
            name="Wafer-B",
            avg_production_time=1.0,
            yield_rate=0.8,
        )
```

### 구현 방향 (GREEN 예고 — RED 단계에서는 작성하지 않음)

- 중복 검사는 저장소의 키 충돌 문제이므로 `SampleRepository.add()`에 둔다:
  이미 존재하는 `sample_id`면 `ValueError`를 발생시키고, 아니면 저장한다.
- `SampleService.register`는 별도 처리 없이 `repository.add()`의 예외를 그대로 전파한다.
- 커스텀 예외 클래스는 아직 두지 않는다(YAGNI) — [1-2]/[1-4]와 동일하게 `ValueError`로 통일.

### 검증 방법

```bash
pytest tests/service/test_sample_service.py -v
```

→ 현재 `SampleRepository.add`에는 중복 검사가 없으므로 `DID NOT RAISE` 실패가 발생하는 것을 확인한다.
