# [Phase 10] 사이클 10-4a — 출고 처리 서브메뉴: CONFIRMED 주문 목록 조회 (보류)

`PLAN.md`의 Phase 10 [10-4a] 항목에 대한 tdd 스킬 사이클 기록.

## 막힘 — 콘솔 메뉴만으로는 CONFIRMED 주문을 만들 수 없다

이 사이클을 [10-1]~[10-3]과 동일하게 `subprocess` 기반 UI 테스트로 설계하려 했으나,
실제로 시나리오를 구성하다 막혔다: 등록 직후 시료 재고는 항상 0이고, 콘솔 메뉴에는
재고를 늘릴 수 있는 명령이 하나도 없다. 재고가 늘어나는 유일한 경로는 생산 완료
(`OrderService.complete_production()`)인데, 이는 [9-2] 검토 당시 "시스템 이벤트로
간주해 메뉴에 노출하지 않는다"고 결정했다. 그 결과 **현재 콘솔 UI만으로는 주문을
`CONFIRMED` 상태로 만드는 것이 원천적으로 불가능하다** — 예약 후 승인하면 재고가
항상 부족(0)해 무조건 `PRODUCING`으로 가고, 그 이후 진행할 메뉴가 없다.

이는 [10-4]/[10-5] 전체(출고 처리, 생산 라인 서브메뉴)에 영향을 주는 더 근본적인
기능 공백이라 판단해, 진행을 멈추고 사람 파트너에게 확인했다.

## 결정

- 생산 라인 메뉴에 **"생산 완료 처리"** 명령을 추가하기로 했다 ([5-7] 당시 결정을
  번복 — `PLAN.md` [10-5c] 참조).
- 진행 순서를 **[10-5c] → [10-4a] → [10-4b]**로 조정한다. [10-5c]를 먼저 구현해
  콘솔에서 예약→승인(PRODUCING)→생산 완료 처리(CONFIRMED)까지 도달 가능하게 만든
  뒤, 이 사이클([10-4a])을 원래 계획대로(콘솔 UI `subprocess` 테스트) 재개한다.

이 문서는 [10-5c] 완료 후 다시 열어 RED부터 진행한다.
