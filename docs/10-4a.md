# [Phase 10] 사이클 10-4a — 출고 처리 서브메뉴: CONFIRMED 주문 목록 조회 (재개)

`PLAN.md`의 Phase 10 [10-4a] 항목에 대한 tdd 스킬 사이클 기록. [10-5c] 완료로 콘솔
UI만으로 `CONFIRMED` 상태에 도달할 수 있게 되어 원래 계획대로 재개한다
(보류 사유는 이 문서의 이전 버전 — git 이력 참고).

## RED

### 목표 (단일 동작)

메인 메뉴에서 `4`를 선택하면 출고 처리 서브메뉴로 진입하고, `1`(목록 조회)을 선택하면
`CONFIRMED` 상태 주문 목록이 콘솔에 출력된다.

### 설계 메모 — 리뷰 발견 사항(2번) 반영

`MenuController`에 `RESERVED` 목록 조회(`list_reserved_orders`)는 있지만 `CONFIRMED`
목록 조회는 없다. [8-3]의 `list_reserved_with_ids`와 동일한 패턴으로
`OrderService.list_confirmed_with_ids()`, `MenuController.list_confirmed_orders()`를
추가해 이 공백을 채운다.

### 작성할 실패 테스트

`tests/test_main.py`에 아래 테스트를 추가한다.

```python
def test_출고_메뉴에서_CONFIRMED_목록을_조회하면_고객명이_출력된다():
    user_input = "\n".join(
        [
            "1",  # 시료 관리
            "1",  # 등록
            "SMP-001",
            "Wafer-A",
            "2.5",
            "0.9",
            "0",
            "2",  # 주문
            "1",  # 예약 (재고 0 -> 승인 시 PRODUCING)
            "SMP-001",
            "홍길동",
            "9",
            "3",  # 승인
            "ORD-0001",
            "0",
            "5",  # 생산 라인
            "3",  # 생산 완료 처리 -> CONFIRMED
            "0",
            "4",  # 출고 처리
            "1",  # CONFIRMED 목록 조회
            "0",
            "0",
            "",
        ]
    )

    result = subprocess.run(
        [sys.executable, "main.py"],
        input=user_input,
        capture_output=True,
        text=True,
        cwd=SRC_DIR,
    )

    assert result.returncode == 0
    assert result.stderr == ""
    assert "홍길동" in result.stdout
```

### 구현 방향 (GREEN 예고 — RED 단계에서는 작성하지 않음)

- `OrderService.list_confirmed_with_ids()`: `list_reserved_with_ids()`와 동일한 패턴으로
  `OrderStatus.CONFIRMED`인 것만 필터링해 `(order_id, order)` 쌍을 반환.
- `MenuController.list_confirmed_orders()`: 위 결과를 기존 `_order_to_dict`로 변환해 반환.
- `ConsoleView.render_release_menu()`: "1. 목록 조회 / 2. 출고 실행 / 0. 뒤로가기".
- `main.py`에 `handle_release_menu(controller, view)`를 추가하고, 메인 루프
  `choice == "4"`에서 호출한다. `1`(목록 조회)만 이번 사이클에서 구현하고
  `2`(출고 실행)는 [10-4b]에서 추가한다.
