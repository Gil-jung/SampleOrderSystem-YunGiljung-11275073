# [Phase 4] 사이클 4-1 — 접수된 주문 목록 조회

`PLAN.md`의 Phase 4 [4-1] 항목에 대한 tdd 스킬 사이클 기록.

## RED

### 목표 (단일 동작)

`OrderService.list_reserved()`가 `RESERVED` 상태인 주문만 반환한다. (PRD.md 4.3절)

### 작성할 실패 테스트

`tests/service/test_order_service.py`에 아래 테스트를 추가한다.

```python
from model.order import Order, OrderStatus


def test_접수된_주문_목록_조회_시_RESERVED_상태만_반환된다():
    order_repository = OrderRepository()
    sample_repository = SampleRepository()
    sample_repository.add(
        Sample(sample_id="SMP-001", name="Wafer-A", avg_production_time=2.5, yield_rate=0.9)
    )
    service = OrderService(order_repository, sample_repository)

    service.reserve(sample_id="SMP-001", customer_name="홍길동", quantity=5)

    rejected_order = Order(sample_id="SMP-001", customer_name="김철수", quantity=3)
    rejected_order.transition_to(OrderStatus.REJECTED)
    order_repository.add(rejected_order)

    reserved_orders = service.list_reserved()

    assert len(reserved_orders) == 1
    assert reserved_orders[0].customer_name == "홍길동"
    assert reserved_orders[0].status == OrderStatus.RESERVED
```

`REJECTED` 주문을 직접 만들어 `order_repository.add()`로 넣는 것은, [1-5]의
`_make_order` 헬퍼와 같은 이유로 테스트 셋업(전제 조건 구성)이며 안티패턴이 아니다 —
`transition_to`의 정상 API를 사용해 상태를 바꾼 뒤 저장소에 넣는 것이므로 우회가 아니다.

### 구현 방향 (GREEN 예고 — RED 단계에서는 작성하지 않음)

- `OrderRepository.list()`: 저장된 모든 `Order`를 리스트로 반환 (`SampleRepository.list()`와
  동일한 패턴).
- `OrderService.list_reserved()`: `order_repository.list()` 결과에서
  `status == OrderStatus.RESERVED`인 것만 필터링해 반환.

### 검증 방법

```bash
pytest tests/service/test_order_service.py -v
```

→ `OrderRepository.list`, `OrderService.list_reserved`가 아직 없으므로 `AttributeError`로
실패하는 것을 확인한다.
