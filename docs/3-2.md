# [Phase 3] 사이클 3-2 — 존재하지 않는 시료로 예약 거부

`PLAN.md`의 Phase 3 [3-2] 항목에 대한 tdd 스킬 사이클 기록.

## RED

### 목표 (단일 동작)

등록되지 않은 `sample_id`로 `OrderService.reserve`를 호출하면 거부된다(`ValueError`).
(PRD.md 4.1절 — "시스템에 등록된 시료만 주문 가능")

### 설계 메모 — 기존 테스트에 영향

`OrderService`가 시료 존재를 검증하려면 `SampleRepository`를 알아야 한다. 생성자에
`sample_repository`를 필수 인자로 추가한다:

```python
OrderService(order_repository, sample_repository)
```

이 때문에 [3-1]에서 작성한 기존 테스트
(`test_주문_예약_생성_시_RESERVED_상태_주문이_저장된다`)도 함께 손봐야 한다 — 생성자
인자를 맞추고, `SampleRepository`에 `SMP-001`을 실제로 등록해둬야 새 검증 로직에
걸리지 않는다. 이는 새로운 동작 추가가 아니라 기존 테스트를 새 생성자 시그니처에
맞추는 필수 조정이므로 사이클 범위를 벗어나지 않는다.

### 작성할 실패 테스트

`tests/service/test_order_service.py`

```python
import pytest

from model.order import OrderStatus
from repository.order_repository import OrderRepository
from repository.sample_repository import SampleRepository
from service.order_service import OrderService


def test_주문_예약_생성_시_RESERVED_상태_주문이_저장된다():
    order_repository = OrderRepository()
    sample_repository = SampleRepository()
    sample_repository.add(
        Sample(sample_id="SMP-001", name="Wafer-A", avg_production_time=2.5, yield_rate=0.9)
    )
    service = OrderService(order_repository, sample_repository)

    order_id = service.reserve(sample_id="SMP-001", customer_name="홍길동", quantity=5)

    stored = order_repository.get(order_id)
    assert stored.sample_id == "SMP-001"
    assert stored.customer_name == "홍길동"
    assert stored.quantity == 5
    assert stored.status == OrderStatus.RESERVED


def test_존재하지_않는_시료로_예약하면_거부된다():
    order_repository = OrderRepository()
    sample_repository = SampleRepository()
    service = OrderService(order_repository, sample_repository)

    with pytest.raises(ValueError):
        service.reserve(sample_id="UNKNOWN", customer_name="홍길동", quantity=5)
```

(`Sample` import 추가 필요)

### 구현 방향 (GREEN 예고 — RED 단계에서는 작성하지 않음)

- `OrderService.__init__(self, order_repository, sample_repository)`로 변경.
- `reserve()` 시작 부분에서 `sample_repository.get(sample_id)`를 호출해 존재를 확인한다.
  `SampleRepository.get`은 없는 키에 `KeyError`를 던지므로, 이를 잡아 `ValueError`로
  변환한다 (Service 계층은 Repository의 저수준 예외를 그대로 노출하지 않는다).

### 검증 방법

```bash
pytest tests/service/test_order_service.py -v
```

→ 신규 테스트는 `TypeError`(생성자 인자 불일치) 또는 `DID NOT RAISE`로 실패하는 것을 확인한다.
