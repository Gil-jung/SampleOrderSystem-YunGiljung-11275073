# [Phase 10] 사이클 10-1b — 시료 관리 서브메뉴: 검색

`PLAN.md`의 Phase 10 [10-1b] 항목에 대한 tdd 스킬 사이클 기록.

## RED

### 목표 (단일 동작)

시료 관리 서브메뉴에서 `3`(검색)을 선택하고 검색어를 입력하면, 검색어에 부분
일치하는 시료만 콘솔에 출력된다.

### 설계 메모

[10-1a]의 GREEN 구현에서 `handle_sample_menu`의 `elif choice == "3":` 분기는 이미
작성해 두었다 (`controller.search_samples(keyword)` 호출 + `view.render_sample_list`
출력). 그러나 이 분기를 실제로 실행하는 테스트는 아직 없었다 — [10-1a]의 테스트는
등록(`1`)과 전체 조회(`2`)만 검증했다. 이번 사이클은 이미 작성된 코드가 실제로
검색 기능을 올바르게 수행하는지 **처음으로 테스트**한다.

기존 구현이 이미 존재하므로, 검색어와 일치하지 않는 시료가 결과에서 빠지는지까지
확인해야 진짜 검증이 된다 (단순히 등록한 시료 이름이 출력되는지만 보면 [10-1a]와
차이가 없다). 검색 필터링 자체가 아직 한 번도 실행된 적 없는 코드 경로이므로, 만약
버그가 있다면(예: 오타, 잘못된 메서드 호출) 이 테스트가 실패로 드러낼 것이다 — 즉
"이미 통과가 보장된" 사이클이 아니라 진짜 RED 가능성이 있는 사이클이다.

### 작성할 실패(가능성이 있는) 테스트

`tests/test_main.py`에 아래 테스트를 추가한다.

```python
def test_시료_관리_메뉴에서_검색어로_필터링된_결과만_표시된다():
    user_input = "\n".join(
        [
            "1",  # 메인 메뉴 -> 시료 관리
            "1",  # 등록: Wafer-A
            "SMP-001",
            "Wafer-A",
            "2.5",
            "0.9",
            "1",  # 등록: Other-B
            "SMP-002",
            "Other-B",
            "1.0",
            "0.8",
            "3",  # 검색
            "Wafer",
            "0",  # 시료 관리 메뉴에서 뒤로가기
            "0",  # 메인 메뉴 종료
            "",
        ]
    )

    result = subprocess.run(
        [sys.executable, "main.py"],
        input=user_input,
        capture_output=True,
        text=True,
        cwd=SRC_DIR,
    )

    assert result.returncode == 0
    assert result.stderr == ""
    assert "Wafer-A" in result.stdout
    assert "Other-B" not in result.stdout
```

### 구현 방향 (GREEN 예고)

- [10-1a]에서 이미 작성된 `elif choice == "3":` 분기가 이 테스트를 통과시키는지
  먼저 실행해 확인한다. 통과하면 추가 프로덕션 코드 변경 없이 REVIEW로 넘어간다.
  실패하면(예: 필터링이 실제로 안 되는 버그가 있다면) 그때 수정한다.
