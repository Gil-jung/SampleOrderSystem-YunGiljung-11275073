주문 서브메뉴에서 `2`(접수된 목록 조회)를 선택하면 `RESERVED` 상태 주문 목록이

### 작성할 실패 테스트

`tests/test_main.py`에 아래 테스트를 추가한다.

```python
def test_주문_메뉴에서_접수된_목록을_조회하면_고객명이_출력된다():
    user_input = "\n".join(
        [
            "1",  # 시료 관리
            "1",  # 등록
            "SMP-001",
            "Wafer-A",
            "2.5",
            "0.9",
            "0",
            "2",  # 주문
            "1",  # 예약
            "SMP-001",
            "홍길동",
            "5",
            "2",  # 접수된 목록 조회
            "0",
            "0",
            "",
        ]
    )

    result = subprocess.run(
        [sys.executable, "main.py"],
        input=user_input,
        capture_output=True,
        text=True,
        cwd=SRC_DIR,
    )

    assert result.returncode == 0
    assert result.stderr == ""
    assert "홍길동" in result.stdout
```

### 구현 방향 (GREEN 예고 — RED 단계에서는 작성하지 않음)

- `ConsoleView.render_order_list(orders)`: `list_reserved_orders()`가 반환한 `dict`
  리스트를 사람이 읽을 문자열로 렌더링 (비어있을 때 처리 포함).
- `main.py`의 `handle_order_menu`에 `elif choice == "2":` 분기를 추가해
  `controller.list_reserved_orders()`를 `view.render_order_list(...)`로 출력한다.

### 검증 방법

```bash
pytest tests/test_main.py -v
```

→ `main.py`가 아직 `"2"` 분기를 처리하지 않아(현재는 무시) `assert "홍길동" in result.stdout`가
실패하는 것을 확인한다.
