# SampleOrderSystem — 반도체 시료 생산주문관리 시스템

## 1. 배경

가상의 반도체 회사 **S-Semi**는 다양한 종류의 반도체 시료(Sample)를 생산하여
연구소, 팹리스(Fabless) 업체, 대학 연구실 등의 고객에게 납품한다.

시료는 주문이 들어오면 웨이퍼 공정 설비를 통해 제작되고, 검수를 거쳐 고객에게 출고된다.
최근 주문량이 급증하면서 엑셀과 메모장으로 주문을 관리하는 방식의 한계(잦은 실수, 재고·공정
현황 파악 어려움)가 드러났고, 이를 해결하기 위해 **반도체 시료 생산주문관리 시스템**을
개발하기로 결정했다.

## 2. 역할별 흐름도

| 역할 | 책임 |
|---|---|
| **고객** | 필요한 시료를 주문 담당자에게 요청, 출고된 주문을 수령 |
| **주문 담당자** | 고객의 요청에 맞게 주문서 작성, 생산 담당자에게 주문서 전달 |
| **생산 담당자** | 주문 담당자로부터 주문서 수신 후 승인/거절 판단, 생산 라인에 생산 요청, 재고 확인, 출고 준비, 출고 처리 |

## 3. 생산 라인 시스템 개요

- 공장에서 시료 하나를 생산하는 설비 흐름을 다룬다.
- 하나의 생산 라인은 시료를 하나씩 생산한다.
- 주문이 들어온 시료에 대해서만 생산한다.
- 시스템은 **콘솔 기반**으로 동작한다.
- 담당자가 직접 명령을 입력하여 시료를 등록하고 주문을 처리한다.

## 4. 주문 상태 흐름

모든 주문은 아래 상태 중 하나를 가진다. `REJECTED`는 거절된 주문으로 정상 흐름 밖의 상태이며
모니터링 대상에서 제외한다.

| 상태 | 의미 |
|---|---|
| `RESERVED` | 주문 접수 |
| `REJECTED` | 주문 거절 (정상 흐름 외, 모니터링 제외) |
| `PRODUCING` | 주문 승인 완료 및 재고 부족으로 생산 중 |
| `CONFIRMED` | 주문 승인 완료 및 출고 대기 중 |
| `RELEASE` | 출고 완료 |

정상 흐름: `RESERVED` → (`PRODUCING` →) `CONFIRMED` → `RELEASE`
(재고가 충분하면 `PRODUCING`을 거치지 않고 바로 `CONFIRMED`로 진입할 수 있다)

## 5. 참고: PoC 디렉터리

`PoC/`에는 본 시스템에 앞서 개별 기술 요소를 검증한 PoC들이 있다. 각 PoC는 "주문관리
시스템"을 예시 도메인으로 삼아 기술을 검증했을 뿐, 본 CLAUDE.md의 도메인(반도체 시료
생산주문관리)과 상태값·용어가 다를 수 있음에 유의한다.

- `PoC/ConsoleMVC` — 콘솔 기반 MVC 계층 분리 검증
- `PoC/DataMonitor` — 인메모리 데이터 실시간 콘솔 모니터링 검증
- `PoC/DataPersistence` — JSON 파일 기반 CRUD 영속화 검증
- `PoC/DummyDataGenerator` — 테스트용 더미 데이터 생성 도구 검증

향후 기능 구현 및 버그 수정은 `tdd` 스킬을 사용하여 진행한다. 구현 코드를 작성하기 전에
테스트를 먼저 작성하는 TDD(Red→Green) 방식을 따른다.
