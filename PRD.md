# PRD — 반도체 시료 생산주문관리 시스템

배경 및 도메인 개요는 [`CLAUDE.md`](./CLAUDE.md)를 참고한다. 본 문서는 콘솔 기반 애플리케이션의
기능 요구사항을 정의한다.

## 1. 용어 정의

| 용어 | 정의 |
|---|---|
| 시료 (Sample) | 시스템의 가장 기본이 되는 생산/주문 단위. 고유 ID와 속성(이름, 평균 생산시간, 수율)을 가진다. |
| 재고 (Stock) | 현재 출고 가능한 시료의 보유 수량. |
| 주문 (Order) | 고객이 특정 시료를 특정 수량 요청한 건. 상태 흐름을 가진다. |
| 생산 큐 (Production Queue) | `PRODUCING` 상태 주문의 부족분을 생산하기 위해 대기하는 FIFO 큐. |
| 수율 (Yield) | 정상 생산된 시료 수 / 총 생산 시도 수. 예: 100개 생산 중 90개 정상 → 0.9 |

### 주문 상태 (`CLAUDE.md` 4절과 동일)

`RESERVED` → (`PRODUCING` →) `CONFIRMED` → `RELEASE`, 그리고 정상 흐름 밖의 `REJECTED`.

## 2. 데이터 모델

### 2.1 시료 (Sample)

| 필드 | 타입 | 설명 |
|---|---|---|
| `sample_id` | string | 고유 식별자 |
| `name` | string | 시료 이름 |
| `avg_production_time` | number | 개당 평균 생산시간 |
| `yield_rate` | number (0~1) | 수율 = 정상 생산 시료 수 / 총 생산 시료 수 |
| `stock` | int | 현재 재고 수량 (등록 시 0, 생산 완료 시 증가, 출고 시 감소) |

### 2.2 주문 (Order)

| 필드 | 타입 | 설명 |
|---|---|---|
| `order_id` | string | 고유 식별자 |
| `sample_id` | string | 주문 대상 시료 |
| `customer_name` | string | 고객명 |
| `quantity` | int | 주문 수량 |
| `status` | enum | `RESERVED` / `REJECTED` / `PRODUCING` / `CONFIRMED` / `RELEASE` |
| `created_at` | datetime | 예약(접수) 시각 |
| `updated_at` | datetime | 마지막 상태 변경 시각 |

## 3. 메인 메뉴

애플리케이션 실행 시 아래 메뉴를 표시하고, 상단에 **전체 시료에 대한 요약 정보**
(등록 시료 수, 전체 재고 합계 등)를 함께 보여준다.

1. 시료 관리
2. 주문 (접수 / 승인 / 거절)
3. 모니터링
4. 출고 처리
5. 생산 라인

## 4. 기능 요구사항

### 4.1 시료 관리

시스템에 등록된 시료만 주문 가능하다.

- **시료 등록**: `시료 ID`, `이름`, `평균 생산시간`, `수율`을 입력받아 신규 시료를 추가한다.
  초기 재고는 0이다.
- **시료 조회**: 등록된 모든 시료 목록과 각 시료의 **현재 재고 수량**을 함께 표시한다.
- **시료 검색**: 이름 등 속성으로 특정 시료를 검색한다.

### 4.2 시료 주문 (예약)

- 고객이 원하는 시료와 수량으로 주문 담당자가 주문을 생성한다.
- **입력 값**: `시료 ID`, `고객명`, `주문 수량`
- 생성된 주문은 상태 `RESERVED`로 등록된다.

### 4.3 주문 승인 / 거절

- **접수된 주문 목록**: `RESERVED` 상태인 주문 목록을 표시한다.
- **주문 승인**: 특정 `RESERVED` 주문을 승인하면, 재고 상황에 따라 자동으로 분기한다.
  - 재고(`stock`) ≥ 주문 수량(`quantity`) → 즉시 `CONFIRMED`로 전환하고, 재고에서 주문 수량만큼 차감(예약)한다.
  - 재고 < 주문 수량 → 생산 라인의 생산 큐에 자동 등록하고, 주문 상태를 `PRODUCING`으로 전환한다.
- **주문 거절**: 특정 `RESERVED` 주문을 즉시 `REJECTED`로 전환한다. 이후 정상 흐름/모니터링에서 제외한다.

### 4.4 모니터링

- **주문량 확인**: 상태별(`RESERVED` / `CONFIRMED` / `PRODUCING` / `RELEASE`) 주문 수 및 목록을 표시한다.
  `REJECTED`는 유효한 주문이 아니므로 집계에서 제외한다.
- **재고량 확인**: 시료별 현재 재고 수량을 표시하고, 해당 시료의 미출고 주문 수요(=`CONFIRMED` 상태로
  출고 대기 중인 주문 수량의 합) 대비 재고 상태를 아래 기준으로 함께 표기한다.
  - **여유**: 재고가 미출고 수요보다 많음
  - **부족**: 재고가 0보다 크지만 미출고 수요보다 적음
  - **고갈**: 재고가 0

### 4.5 생산 라인

승인 시점에 재고가 부족했던 주문(`PRODUCING`)의 부족분을 생산 큐에서 순서대로 생산한다.

- **부족분**: `주문 수량 - 승인 시점의 가용 재고`
- **실 생산량**: `ceil(부족분 / 수율)` — 수율에 따른 불량(오차)을 감안해 목표 정상 수량을 확보할 수 있도록 여유 있게 생산한다.
- **총 생산 시간**: `평균 생산시간 × 실 생산량`
- 생산이 완료되면 정상 생산분을 재고에 반영하고, 해당 주문 상태를 `PRODUCING` → `CONFIRMED`로 전환한다.
- **생산 현황 표기**: 현재 생산 중인 시료에 대한 정보(예: 대상 주문, 목표 수량, 현재까지 생산량, 예상 완료 시점 등)를 표시한다. 표시 항목의 구체적 수준은 구현 시 자율적으로 결정한다.
- **대기 주문 확인**: 생산 큐에 대기 중인 주문 목록을 **FIFO** 순서로 표시한다.

### 4.6 출고 처리

- `CONFIRMED` 상태 주문 중 특정 주문을 선택해 출고를 실행한다.
- 출고 처리 시 주문 상태를 `RELEASE`로 전환한다.

## 5. 상태 전이 요약

```
RESERVED --승인(재고충분)--> CONFIRMED --출고--> RELEASE
RESERVED --승인(재고부족)--> PRODUCING --생산완료--> CONFIRMED --출고--> RELEASE
RESERVED --거절-------------> REJECTED (종결, 모니터링 제외)
```

## 6. 계산식 정리

| 항목 | 공식 |
|---|---|
| 수율 | 정상 생산 시료 수 / 총 생산 시료 수 |
| 부족분 | 주문 수량 − 승인 시점 가용 재고 |
| 실 생산량 | `ceil(부족분 / 수율)` |
| 총 생산 시간 | 평균 생산시간 × 실 생산량 |

## 7. 확정된 전제 (비범위)

- **생산 라인은 1개만 운영한다** (`CLAUDE.md` 3절과 동일). 생산 큐는 단일 FIFO 큐이며, 동시에 생산 중인
  시료는 최대 1건이다.
- **하나의 주문은 하나의 시료 종류만 요구한다.** 주문 1건 : 시료 1종 : 수량 1개 값으로 구성된다.
- **출고는 주문 단위 전량 처리로 가정한다.** 주문 수량 중 일부만 출고하는 부분 출고는 지원하지 않으며,
  출고 처리 시 해당 주문의 수량 전체가 한 번에 `RELEASE`로 전환된다.
