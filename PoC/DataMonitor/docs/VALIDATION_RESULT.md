## 자동화 테스트 실행 방법
```bash
./.venv/Scripts/python.exe -m pytest tests/ -v
```
전체 31개 테스트 통과 (`store` 5, `order_simulator` 3, `monitor` 15, `scenarios` 8).

## 수동 검증이 필요한 항목

자동화 테스트는 로직의 정합성(유실 없음, 예외 없음, 표시 정확성)을 검증하지만,
아래 항목은 실제 장시간 실행 환경에서 별도로 확인이 필요하다.

- **S4 (실제 동시성 부하)**: 여러 스레드/프로세스에서 동시에 대량 생성 시 락 경합에 따른
  성능 저하 여부. 확인 방법: `order_simulator.py`를 여러 스레드에서 동시 실행 후 소요 시간 측정.
- **S8 (장시간 메모리/CPU)**: `python src/monitor.py --interval 1` 을 수 시간 백그라운드로
  실행하며 작업관리자 또는 `tracemalloc`으로 메모리 증가 추이를 관찰. `main_loop`는 매 반복마다
  `prev_snapshot`을 새 리스트로 교체하므로 이론상 누적 누수는 없으나, 실제 장시간 측정으로
  최종 확인 권장.

## 결론
S1~S8 전 항목이 자동화 테스트 기준으로 PASS 하였다. PoC.md 4.1절 DoD 중
"S1~S8 시나리오 전체 통과"는 자동화 가능한 범위 내에서 충족되었으며,
장시간 실행 리소스 검증은 운영 환경 반영 전 별도 수동 확인을 권장한다.
