# Dummy Data Generator — PoC 계획서

## 1. 개요

테스트에 필요한 Dummy(가짜) 데이터를 손쉽게 생성할 수 있는 **콘솔 기반 도구**를 개발한다.
초기 PoC 단계에서는 "주문관리 시스템" 시나리오를 예시로 삼아 기능을 검증하고,
검증이 끝나면 범용적으로 사용 가능한 완전한 형태의 Dummy 데이터 생성 도구로 발전시킨다.

### 목표
- 테스트 목적의 Dummy 데이터를 반복 가능하고 신뢰성 있게 생성
- 생성 결과는 **JSON 파일**로 저장하여 다른 시스템/테스트 코드에서 바로 활용 가능하도록 함
- 실제 업무 시나리오(주문관리)를 통해 기능의 실효성을 검증
- 검증된 기능을 일반화하여 다양한 도메인에 적용 가능한 완전한 도구로 완성

## 2. 단계별 진행 계획

### 2-1단계. Dummy 데이터 생성 Tool (기본 기능 구현)

콘솔에서 실행되는 CLI 도구로 아래 기능을 제공한다.

- **입력**: 데이터 스키마(필드명, 타입, 제약조건 등) 정의
- **처리**: 정의된 스키마를 기반으로 임의의 값 생성
- **출력**: 생성된 Dummy 데이터를 `.json` 파일로 저장

#### 지원 예정 데이터 타입
| 타입 | 설명 | 예시 |
|---|---|---|
| 문자열 | 이름, 주소, 상품명 등 | `"홍길동"` |
| 숫자 | 수량, 금액, ID 등 | `10000` |
| 날짜/시간 | 주문일시, 배송일 등 | `"2026-07-15T10:00:00"` |
| Boolean | 결제 여부, 취소 여부 등 | `true` / `false` |
| Enum | 상태값(주문상태, 결제상태 등) | `"PAID"`, `"CANCELLED"` |
| 배열/객체 | 리스트, 중첩 구조 | 주문 상품 목록 |

#### CLI 사용 예시 (안)
```bash
dummy-gen generate --schema order-schema.json --count 100 --output orders.json
```

#### 산출물
- 콘솔 실행 파일(스크립트)
- Dummy 데이터 스키마 정의 포맷(JSON Schema 기반)
- 생성된 JSON 데이터 파일

### 2-2단계. 주문관리 시스템 시나리오 기반 검증

실제 사용 사례를 상정하여 도구의 기능을 테스트하고 정확성을 검증한다.

#### 시나리오 예시: 주문 입력 → 연관 데이터 자동 생성

사용자가 아래와 같은 "주문" 정보를 입력하면, 해당 주문과 관련된 데이터를 자동으로 생성한다.

**입력 예시**
```json
{
  "orderId": "ORD-0001",
  "customerName": "홍길동",
  "items": [
    { "productName": "무선 마우스", "quantity": 2, "unitPrice": 15000 }
  ]
}
```

**생성되어야 할 연관 Dummy 데이터**
- 주문 상세(Order Detail): 주문일시, 배송지, 배송 상태
- 결제 정보(Payment): 결제수단, 결제금액, 결제상태
- 고객 정보(Customer): 고객 ID, 연락처, 등급
- 배송 정보(Shipment): 운송장 번호, 배송 상태 이력

**출력 예시**
```json
{
  "order": {
    "orderId": "ORD-0001",
    "orderDate": "2026-07-15T09:30:00",
    "status": "PAID",
    "customer": { "customerId": "CUST-1024", "name": "홍길동", "phone": "010-1234-5678" },
    "items": [
      { "productName": "무선 마우스", "quantity": 2, "unitPrice": 15000, "totalPrice": 30000 }
    ],
    "payment": { "method": "CARD", "amount": 30000, "status": "COMPLETED" },
    "shipment": { "trackingNo": "TRK-998877", "status": "PREPARING" }
  }
}
```

#### 검증 항목
- [ ] 입력값이 결과 데이터에 정확히 반영되는가
- [ ] 연관 필드(고객/결제/배송) 간 정합성이 유지되는가 (예: 결제금액 = 수량 × 단가 합)
- [ ] 동일 입력에 대해 반복 실행 시 재현 가능한 결과를 낼 수 있는가(옵션: 시드 고정)
- [ ] 다량 생성 시(N건) 성능과 데이터 유일성(ID 중복 없음) 확인
- [ ] 잘못된 입력(누락 필드, 타입 불일치)에 대한 예외 처리 확인

### 2-3단계. 완전 무결한 Dummy 데이터 생성 Tool 완성

주문관리 시나리오 검증을 통해 발견된 문제를 보완하고, 아래 요건을 만족하는 최종 버전을 완성한다.

- **정합성**: 연관 데이터 간 참조/계산 값이 항상 논리적으로 일치
- **재현성**: 시드(seed) 지정 시 동일한 데이터 재생성 가능
- **확장성**: 주문관리 외 다른 도메인 스키마도 손쉽게 추가 가능한 구조
- **검증 자동화**: 생성 데이터에 대한 자동 유효성 검사(스키마 검증) 내장
- **문서화**: 스키마 작성 가이드 및 CLI 사용법 문서 제공
